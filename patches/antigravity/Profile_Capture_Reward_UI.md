# 积分盲盒福利与首登画像问卷优化

## 核心更新点
本次升级彻底移除了原有“生硬采集用户画像”的弹窗广告感，并依托 Zibll 的经济系统引入了首单绑定福利空投机制。

1. **前端视觉与文案优化（去“牛皮癣”化）**：
   - 将原先突兀的紫色高光弹窗、以及“为了给你提供更精准的内容请做调查”等容易引起用户反感的采集话术彻底铲除。
   - 重构为极简柔和的**专属金橙色双行横幅**（`#ff7a00` 色系），支持自适应透明度暗黑模式，呈现“盲盒彩蛋已就绪”的福利氛围。
   - 底部操作按钮文案根据选择进度，巧妙切换至“🎁 开启神秘盲盒并完成”。

2. **问卷限制与交互调优**：
   - 彻底梳理 JavaScript 验证逻辑：修正了维度 3（消费习惯）的错误显示和提交下限限制问题。
   - 通过强硬的话术引导：“已选 0 / 至少选 N 个” 来倒逼用户产生正确选项（而非原来随意的“支持多选”字眼），防止随意跳过。
   - 解决了 DOM 操作中偶发的弹窗刷新/多节点注入污染问题，采用了精准选择器。

3. **后端闭环防刷空投逻辑 (`user-profile.php`)**：
   - 利用钩子 `xingxy_capture_profile_on_bind`，在验证用户成功输入验证码并在 Zibll 建立绑定关系的同时触发拦截。
   - **防薅羊毛（Idempotency Key）**：依赖于对 User Meta `_xingxy_welcome_rewarded` 的读写死锁机制，保证每个用户终生仅享受一次“新人漫游礼包”。
   - **静默入账与洗白**：调用底层 `zibpay_update_user_points`，下发 150 积分，流水抬头包装为 **《🎁 星星球首次探索漫游奖励！(神秘盲盒)》**。坚决不向任何前台界面泄露“测试性格/填表奖励”的初衷，沉浸感拉满。
