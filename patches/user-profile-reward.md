# 新人盲盒奖励与用户测试画像系统

## 1. 核心目标
为了提升用户绑定邮箱的比率以及完善用户基础画像数据，在 Zibll 主题原生的“绑定邮箱/手机号”弹窗流程中，强行切入了一组“首单盲盒奖励”问卷问答流。
以“新人福利”的名义引诱用户完成我们预设的测试维度，收集其喜好标签（明星、游戏、消费破冰），并在完成后自动发放 150 积分存入其站内账户。

## 2. 涉及文件与修改点总览

### 后端与后台设置 (PHP)
- **`inc/options.php`**: 
  新增了整个 `用户画像收集` 的 CSF 后台设置面板。允许管理员动态配置三个维度的题目图片、标题文案以及关联选项。
- **`inc/user-profile.php`** (新文件):
  提供核心的钩子函数 `xingxy_ajax_get_profile_options` 供前端异步拉取题目。
  挂载在 Zibll 原生 `zib_user_bind_email` 和 `zib_user_bind_phone` 端点，负责在中途劫持表单提交过来的 `dim1, dim2, dim3` 选择序列，将多选 ID 落表到用户 Meta，并**自动调用 `zibpay_update_user_points` 分发 150 积分盲盒奖励**。
  包含严格的防刷机制：只有第一次绑定的用户在 `_xingxy_welcome_rewarded` 枚举不存在时才予以下放，且账单备注精装为 `[盲盒开启] 🎁 星星球首次探索漫游奖励！(神秘盲盒)`，不暴露“画像收集”的内部用途。
- **`inc/assets.php`**: 
  负责在前台局部注入专属于这套流程的 Js/Css 依赖。
- **`init.php`**: 
  挂载/引入 `user-profile.php`。

### 前端逻辑与表现 (JS / CSS)
- **`assets/js/profile-capture.js`** (新文件):
  负责监听 Zibll 原生弹窗的“发送验证码”倒计时动作，一旦倒计时开始，即动态获取选项数据，并在原生表单内部插入“分布滑屏式”的问卷 HTML 结构。
  控制“下一步”跳转，并自动阻断原生的 Zibll `submit` 禁用解除动作，直至用户完成了 `reqLimits`（目前规定：维度一可多选无需下限，维度二2个，维度三必须1个及以上）。
  最后将最终提交按钮文案精装为：“🎁 开启盲盒并完成”，大幅度提升用户提交积极性。
- **`assets/css/profile-capture.css`** (新文件):
  搭配呈现高级视觉重载。顶部包含一段适配暗色与浅色模式的“金橙色虚线盲盒横幅”，文字内容明确指向福利和抽奖。

## 3. 安装与卸载注意事项
- 禁用此功能只需要在 `init.php` 中注释掉引入 `user-profile.php` 并移除对应的 JS/CSS Hook 即可恢复原生 Zibll 体验。
- 如需更改单维度的必定选择下限数量，请编辑 `assets/js/profile-capture.js` 中的 `reqLimits` 变量。
- 对于 ZIbll 主题原本附带的 `zibsvg-` 开头图标集问题，代码中采用了深度 `zib_svgs` 挂载取用机制，以防其外部 sprite SVG 文件未装载完整。
