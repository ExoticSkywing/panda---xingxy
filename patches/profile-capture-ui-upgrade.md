# 用户画像采集端 - UI 视觉重构与盲盒特效升级

## 问题背景 (Problem Background)
在前期上线的“绑定邮箱领盲盒”功能中，前端界面存在以下几个主要痛点：
1. **视觉割裂感严重**：原始版的福利横幅（金橙色卡片）和辅导类提示框由于硬编码了较多鲜艳的底色和光晕，在 Zibll 系统切换为“夜间深色模式”时显得格格不入，像一块强行贴上去的补丁。
2. **缺乏即时正反馈**：用户在完成所有选项并点击“开启盲盒”时，只有干巴巴的弹窗跳转，缺乏获得福利的仪式感和“彩蛋”该有的情绪释放。
3. **Zibll 绑定提醒文案过时**：由于近期邮箱服务商规则变化，验证邮件进垃圾箱的概率增加，系统需要一个更强力、更具引导性且适配原生深浅模式的绑定提醒文案。

## 修复排查过程 (Debugging Process)
1. **沙盒式 UI 单元测试**：为了不影响线上用户的正常闭环，在 `profile-capture.js` 的独立外挂层（右下角悬浮窗）构建了多套新版 UI 进行灰度预览。不断剥离硬编码（Hard-coded）的复杂色彩代码。
2. **底层原生化提取**：放弃自定义颜色，将新 UI 全部重构为调用 Zibll 原生 CSS 变量（如 `var(--muted-bg)`、`var(--theme-color)`、`var(--main-color)`），确保新组件能随全局主题一键切换日/夜间模式而自动完美融入。
3. **特效引擎沙盒验证**：同样通过“沙盒测试”的方式引入了 `canvas-confetti` 库，先挂载在独立按钮上供管理员调配“烟花”的爆发粒度、散射角度与 Z 轴层级 (`zIndex`)，避免正式上线被其他弹窗遮挡。

## 解决方案 (Solution)

### 1. 全新原生风福利横幅 (`assets/css/profile-capture.css`)
- 完全重构了 `.xingxy-native-banner`。摒弃了原来的橙色大方框，改用环境光隐身结构。
- 外层卡片、边框、主文本全部采用 `var()` 变量自动匹配日、夜模式。
- 仅保留了代表福利的 `[彩蛋]` 标签和高亮说明文字使用极光橙渐变（`#ff8c00` ~ `#ff5e00`），实现“原生融合感”与“点亮视觉落点”的巧妙平衡。

### 2. 实装开启彩蛋烟花 (`assets/js/profile-capture.js`)
- 在前端资源栈引入业界标杆抛洒物引擎：`confetti.browser.min.js` (1.9.2)。
- 编写 `xingxyPlayConfetti()` 多段发射逻辑函数。设置超高 `zIndex: 9999999`。
- 将特效精准挂载至原 Zibll 表单的 `button.signsubmit-loader` 提交事件上，且只在按钮未被拦截禁用时触发。
- 完成沙盒测试后，彻底销毁了遗留在 `$('body').append` 中的预览用测试代码。

### 3. Zibll 原生绑定弹窗重构 (HTML/CSS)
- 抛弃了呆板卡片式设计，给出一套带**折叠互动 (`details` 标签)** 的全新提示系统。
- 上半区（不可见区）：通过极具警示味的红色微章 `[提醒]`，直出“找回账号唯一凭证”的防封核心目的。
- 下半区（非干扰互动区）：利用 CSS 原生折叠实现“收不到验证码怎么办？”的折叠面板。
- 加入了黑科技防翻车处理与“星际未知电波干扰”的高逼格向导文案。

## 测试方法 (Verification)
1. **原生化测试**：前台打开“绑定邮箱”触发小芽精灵，点击网站右上角的日/夜间模式切换，观察盲盒的福利头部卡片是否完美自动变色自适应。
2. **特效测试**：填写全部选项并倒数完毕，点击主提交按钮，是否从屏幕中下方瞬间喷发漫天散落的交互花瓣彩纸，且完全遮盖住任何弹窗。
3. **垃圾箱提示测试**：进入 Zibll 后台 `用户&互动` -> `手机/邮箱绑定`，粘贴最新的 HTML 分层文案代码，到前台弹窗内查看原生的折叠展开体验及图文规整度。
